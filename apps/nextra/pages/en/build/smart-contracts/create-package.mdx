# Create Package

<Callout>
We recommend installing the Aptos CLI before beginning.
If you haven't already installed the Aptos CLI, see the [CLI section](../cli)
</Callout>

<Steps>

### `aptos move init`

In a new project directory, initialize a Move package by running:

```bash filename="Terminal"
aptos move init --name <PROJECT_NAME>
```

You should now have a Move project that looks like so:

<FileTree>
  <FileTree.Folder name="scripts" defaultOpen></FileTree.Folder>
  <FileTree.Folder name="sources" defaultOpen></FileTree.Folder>
  <FileTree.Folder name="tests" defaultOpen></FileTree.Folder>
  <FileTree.File name="Move.toml" />
</FileTree>

<Callout type={"default"}>
  You can also create a Move package from a [template](../cli/start-from-template).
</Callout>

### Update `Move.toml`

In `Move.toml`, fill in the following key information:

1. `name`: name of your package
2. `version`: package version (default is `"0.0.0"`)
3. `addresses`: Describes which address the module will be deployed to.  These are named addresses that can be used as aliases.  In the below example, we will use `hello_blockchain` as the named address.
4. `dependencies`: You will likely want to use `AptosFramework` and other [Third Party Dependencies](./third-party-dependencies)

Below is an example

```toml filename="Move.toml"
[package]
name = "Examples"
version = "0.0.0"

[addresses]
hello_blockchain = "_"

[dependencies.AptosFramework]
git = "https://github.com/aptos-labs/aptos-framework.git"
rev = "mainnet"
subdir = "aptos-framework"
```
