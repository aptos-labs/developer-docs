# 运行验证者和验证者全节点(VFN)

如果您想参与 Aptos 的共识协议，您需要部署并运行一台验证节点，也可以选择性地部署一台验证器全节点（VFN）。本文档为您概述了部署这两种类型节点所必须的重要步骤。

<Callout type="info">
虽然 VFN 不是参与共识所必需的，但强烈建议每个验证节点的运营者运行一个 VFN。

这是因为 VFN 是区块链数据连接生态系统的唯一入口和出口。网络中拥有较多的 VFN 有助于提高网络可靠性、增加数据可用性，并为大众提供高质量的区块链访问体验。

</Callout>

在初始化质押池或委托池之前，请阅读有关 [质押](../blockchain/staking.mdx) 和 [委托质押](../blockchain/delegated-staking.mdx) 的信息，了解质押池类型之间的区别。请注意，一旦创建了质押池，就不能更改为委托池，反之亦然。另见 [外部资源](../../community/external-resources.md)，了解其他节点运营者提供的工具和服务。

使用本节中的文档来运行 Aptos 验证者和 VFN。在高级层面上，流程如下：

<Steps>
### 理解要求和部署类型

请先阅读节点的性能要求，以便确认所需的计算能力、内存大小、网络带宽和存储空间。然后，选择合适的节点部署方式，可以是云端托管的 Kubernetes、使用 Docker 容器，或者直接通过源代码来配置。

### 为你的节点生成身份

为你的验证者和 VFN 创建公钥 / 私钥对和账户地址。记得不要泄漏你的私钥！

### 配置你的验证者和 VFN

通过 YAML 文件，按照前一步骤生成的密钥和身份信息来配置您的节点。这样做是为了确保您的节点能够安全地与其他节点（也就是对等节点，peer）建立连接。

### 为你的节点下载创世区块和路标文件（genesis and a waypoint）

使用创世文件和路标引导节点，并为启动做准备。
节点配置正确后，安装二进制文件并下载创世块和路标文件。
这些将为你的节点提供它们需要的信息，以便开始与其他节点同步。

先通过创世区块（genesis）和路标文件（waypoint），为启动节点做准备。一旦节点配置无误，接下来安装必要的程序并下载创世区块和路标文件。这将为节点提供必要的信息，以启动并开始与网络中的其他节点同步数据。

### 加入验证者集合并开始参与

首先初始化质押池，然后加入验证者集合，让你的节点开始验证操作，从而获得奖励。在你的节点能够被其他节点接纳之前，你必须确保它加入了验证者集合。这需要你初始化一个质押池，并委托给运营者（operator）和投票者（voter）。质押池准备就绪后，你的节点便可正式加入验证者集合。这标志着你的节点开始与全网同步，你的验证器（validator）开始参与共识决策流程，进而开始获取奖励。

</Steps>
