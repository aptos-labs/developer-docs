# 以太坊到Aptos速查表

要了解更多差异和相似之处，请参阅 [Aptos Learn](https://learn.aptoslabs.com/en/tutorials/ethereum-to-aptos-guide/cheat-sheet?workshop=eth-to-aptos)

| -------------------------- | ------------------------------------------------------------------------------------------------------------------------------ | ------------------------------------------------------------------------------------------------------------------------------------- |
| **智能合约**              | Solidity, EVM                                                                                                                  | Move, MoveVM                                                                                                                          |

| **交易费用**              | 波动大，可能较高                                                                                                          | 费用更低且更稳定                                                                                                            |

| **账户结构**              | 单一余额字段，使用nonce计数                                                                                          | 模块化资源结构，使用序列号                                                                                           |
| **数据存储**              | Patricia默克尔树                                                                                                          | 基于资源和模块的全局存储                                                                                             |
| **存储理念**              | 基于合约的存储                                                                                                         | 以账户为中心的代码与数据存储理念                                                                                             |
| **示例代码**              | [ERC-20](https://github.com/OpenZeppelin/openzeppelin-contracts/tree/master/contracts/token/ERC20)                             | [可互换资产](https://github.com/aptos-labs/aptos-core/blob/main/aptos-move/framework/aptos-framework/sources/fungible_asset.move) |
| **调用方标识**            | `msg.sender`er`                                                                                                                   | `&signer` 引用                                                                                                                   |
| **可升级性**         | 代理模式                                                                                                                 | 直接模块升级                                                                                                                |
| **安全性与保障**      | 易受重入攻击等漏洞影响                                                                                          | 缓解常见安全漏洞                                                                                                      |
| **调度类型**          | 动态调度                                                                                                               | 静态调度                                                                                                                       |
| **同质化代币标准**            | [ERC-20](https://docs.openzeppelin.com/contracts/4.x/erc20)                                                                    | [Coin](../smart-contracts/aptos-coin.mdx) (传统) 和 [可替代资产](../smart-contracts/fungible-asset.mdx)  |
| **非同质化代币标准**          | [ERC-721](https://docs.openzeppelin.com/contracts/4.x/erc721), [ERC-1155](https://docs.openzeppelin.com/contracts/4.x/erc1155) | [数字资产](../smart-contracts/digital-asset.mdx)                                                                      |

### 代币标准详细对比|                        | Solidity                                                          | Move (Aptos)                                                                                                                                                    |
| ---------------------- | ----------------------------------------------------------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **代币结构**          | 每个代币都是独立的合约。                                         | 所有代币均采用统一、可复用的合约，通过类型化的`Coin`或`FungibleAsset`实现。                                                                                     |
| **代币标准**          | 需符合ERC20等标准；具体实现可能不同。                            | 所有代币采用统一的接口和实现方式。                                                                                                                              |
| **余额存储**          | 余额通过映射结构存储在合约中。                                   | **资源导向型余额**：余额作为资源存储在用户账户中。资源无法随意创建，确保代币价值的完整性。                                                                      |
| **转账机制**          | 代币可在无需接收方明确授权的情况下转移。                         | 除特殊情况（如AptosCoin）外，代币转账通常需要接收方的`signer`授权。                                                                                             |

### EVM与Move VM详细对比

- **EVM**：以灵活性和动态调度著称，支持广泛的智能合约行为。但这种灵活性可能导致并行执行和网络操作的复杂性。
- **Move VM**：注重安全性和效率，采用虚拟机与编程语言深度集成的设计。其数据存储模型支持更好的并行化，静态调度方法增强了安全性和可预测性。

<br />
|                              | EVM (以太坊虚拟机)                                               | Move VM (Move虚拟机)                                                                                     |
| ------------------------------- | ---------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------ |
| **数据存储**                    | 数据存储在智能合约的存储空间中。                                     | 数据分布在智能合约、用户账户和对象之间存储。                                                                     |
| **并行化**                      | 由于共享存储空间，并行执行能力受限。                                 | 灵活的分离式存储设计支持更高效的并行执行。                                                                       |
| **虚拟机与语言集成**            | EVM与智能合约语言(如Solidity)是分离的层级。                          | VM层与Move语言无缝集成，原生功能使用Rust编写但可在Move中执行。                                                   |
| **关键网络操作**                | 网络操作的实现可能较为复杂且不够直接。                               | 验证器集管理等关键操作原生实现于Move，支持直接执行。                                                             |
| **函数调用**                    | 动态分发支持任意的智能合约调用。                                     | 静态分发机制更注重安全性和可预测行为。                                                                           |
| **类型安全**                    | 合约类型提供基础的类型安全性。                                       | Move的模块结构和泛型提供更强大的类型安全保障。                                                                   |
| **交易安全**                    | 使用nonce机制保证交易顺序和安全。                                    | 使用序列号(sequence numbers)保证交易顺序和安全。                                                                  |
| **认证存储**                    | 支持，通过智能合约存储实现。                                         | 支持，利用Move的资源模型实现。                                                                                    |
| **对象可访问性**                | 对象不具备全局可访问性，仅限于智能合约作用域内。                     | 保证对象的全局可访问性。                                                                                          |
