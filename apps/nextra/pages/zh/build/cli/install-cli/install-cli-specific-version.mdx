# 安装特定版本的 Aptos CLI (高级)

如果您需要特定版本的 Aptos CLI，您可以直接从 Aptos 源代码构建它。这种安装方法主要用于与 Devnet 上的特定功能交互，这些功能可能还没有在 Testnet / Mainnet 上推出。如果您运行的架构与现有的发布/预编译的二进制文件不兼容，您可能需要按照这些步骤操作。

如果您不需要这种高级方法，您可以在[这里](../../cli.mdx)找到普通安装步骤。

## 在 macOS / Linux 上安装

<Steps>
### 按照步骤[从源代码构建 Aptos](../../../network/nodes/building-from-source.mdx)
### 按照[教程](https://doc.rust-lang.org/cargo/getting-started/installation.html#install-rust-and-cargo)确保您安装了 `cargo`
### 构建 CLI 工具：`cargo build --package aptos --profile cli`
   可用的二进制文件将在 `target/cli/aptos`
### （可选）将此可执行文件移动到您 PATH 中的某个位置。
### 通过运行 `target/cli/aptos help` 来验证安装是否成功。
   这些帮助指令也是用于特定命令的详细指南。
</Steps>

## 在 Windows 上安装

<Steps>
### 按照步骤[从源代码构建 Aptos](https://aptos.dev/guides/building-from-source/)。
### 按照[教程](https://doc.rust-lang.org/cargo/getting-started/installation.html#install-rust-and-cargo)确保您安装了 `cargo`
### 构建CLI工具：`cargo build --package aptos --profile cli`。
   可用的二进制文件将在 `target\cli\aptos.exe`
### （可选）将此可执行文件移动到您 PATH 中的某个位置。
### 通过运行 `target\cli\aptos.exe help` 来验证安装是否成功。
   这个帮助指令也能够用于查看特定命令的详细使用指南。
</Steps>
