# スナップショットからのブートストラップ

このドキュメントでは、スナップショットを使用してAptosノードを素早くブートストラップする方法について説明します。これは、バリデーター、VFN、PFNなど、すべてのノードタイプで実行できます。

<Callout type="warning">
**インデクサースナップショット**<br />
コミュニティが提供するスナップショットは、インデクサーのサポートを提供していません。インデクサーノードをブートストラップする場合は、[バックアップ](./aptos-db-restore.mdx)を使用する必要があります。
</Callout>

[ステートシンク](../configure/state-sync.mdx)を使用して新しいノードをブートストラップすることもできますが、ネットワークが長期間稼働している場合、これは最速のアプローチではないかもしれません。時間がかかりすぎたり、他のノードがすでに台帳履歴を削除している可能性があるため、必要なデータをすべて取得できない場合があります。これを避けるため、既存のノードのストレージデータのコピーである既存のスナップショットを使用してノードをブートストラップすることができます。

<Callout type="warning">
**メインネットのスナップショット**<br />
**メインネット**でノードをブートストラップするためにスナップショットを使用することは推奨されません。これは、スナップショットがAptosソフトウェアによって検証されないためです。その結果、スナップショットが無効であったり、不正確なデータを含んでいる可能性があります。セキュリティ上の懸念を防ぐため、スナップショットは**デブネット**や**テストネット**などのテスト環境でのみ使用することを推奨します。

**メインネット**のスナップショットを使用してノードをブートストラップする必要がある場合は、自身でスナップショットを生成するか、信頼できるソースから入手する必要があります。また、使用する前にスナップショットの整合性と信頼性を確認する必要があります（暗号署名やチェックサムなどを使用）。
</Callout>

## 既存のスナップショットを見つける

さまざまなAptosコミュニティメンバーからダウンロードできるスナップショットがいくつかあります：

- BWareLabs（テストネットとメインネット）：[BWareLabs Aptos Node Snapshots](https://bwarelabs.com/snapshots/aptos)
- Polkachu（メインネット）：[Polkachu Aptos Node Snapshots](https://polkachu.com/aptos_snapshots/aptos)

<Callout type="info">
**スナップショットデータに関する質問** <br />
スナップショットの構築方法と圧縮方法によって、スナップショットファイルのサイズが異なる場合があります。スナップショットデータについて質問がある場合や問題が発生した場合は、[Aptos Discord](https://discord.gg/aptosnetwork)の[#node-support](https://discord.com/channels/945856774056083548/953421979136962560)チャンネルで直接Aptosコミュニティメンバーにお問い合わせください。
</Callout>

## スナップショットを使用する

スナップショットを使用するには、ファイルをダウンロードしてノードのストレージデータベースの場所にコピーするだけです。
この場所は、フルノードの`yaml`設定ファイルの`data_dir`で確認および更新できます。
データディレクトリの設定方法については、例のチュートリアル（[PFNを実行する](../full-node/deployments.mdx)）を参照してください。
