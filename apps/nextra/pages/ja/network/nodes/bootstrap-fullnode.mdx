# 履歴データを使用したノードのブートストラップ

新しいノードのブートストラップは通常、高速同期モードの[ステート同期](configure/state-sync.mdx)のみに依存します。これは、ピアの助けを借りて最新の状態に同期し、そこから最近の台帳履歴の同期を開始します。ただし、何らかの理由でより長い台帳履歴が必要な場合は、以下の選択肢があります：

- ### [スナップショットからのブートストラップ](bootstrap-fullnode/bootstrap-fullnode.mdx)
- ### [バックアップからのブートストラップ](bootstrap-fullnode/aptos-db-restore.mdx)
