# フェデレーテッドキーレスのよくある質問

## 一般的な質問

### フェデレーテッドキーレスとは何ですか？

フェデレーテッドキーレスは、Aptos Keylessの拡張機能で、より多くのOpenID Connect（OIDC）プロバイダーとIAMプロバイダー（Auth0やAWS Cognitoなど）をサポートします。これにより、dAppは既存のアカウントシステムとの互換性を維持しながら、より多くの認証方法を提供できます。

### フェデレーテッドキーレスはどのように機能しますか？

フェデレーテッドキーレスは、IAMプロバイダーを通じてユーザーを認証し、JWTトークンを発行します。このトークンは、ユーザーのブロックチェーンアカウントを導出し、トランザクションを署名するために使用されます。詳細については、[フェデレーテッドキーレスの仕組み](./how-federated-keyless-works)を参照してください。

### どのIAMプロバイダーがサポートされていますか？

現在、以下のIAMプロバイダーがサポートされています：
- Auth0
- AWS Cognito

サポートされているプロバイダーの完全なリストは、[OIDCサポート](../oidc-support)を参照してください。

## セットアップと設定

### フェデレーテッドキーレスをどのように設定しますか？

1. IAMプロバイダーでプロジェクトを設定します
2. JSON Web Key Set（JWKS）をブロックチェーンに登録します
3. フェデレーテッドキーレスプロバイダーを初期化します

詳細な手順については、[統合ガイド](./integration-guide)を参照してください。

### JWKSの登録は必要ですか？

はい、フェデレーテッドキーレスアカウントを使用するには、JWKSをブロックチェーンに登録する必要があります。これは、トランザクションの署名を検証するために必要です。

### キーローテーションはどのように処理しますか？

IAMプロバイダーがキーをローテーションする場合、新しいJWKSをブロックチェーンに登録する必要があります。これを行わないと、ユーザーはトランザクションを送信できなくなります。詳細については、[重要な考慮事項](./key-considerations)を参照してください。

## セキュリティとプライバシー

### フェデレーテッドキーレスは安全ですか？

はい、フェデレーテッドキーレスは以下の機能を提供することで、高いセキュリティを確保しています：
- 暗号化された通信
- セキュアな鍵管理
- トランザクションの署名検証

### ユーザーのプライバシーはどのように保護されますか？

フェデレーテッドキーレスは以下の方法でユーザーのプライバシーを保護します：
- ブロックチェーン上でユーザーの識別情報を隠す
- 最小限のデータのみを保存
- エンドツーエンドの暗号化

### JWKSの所有者アカウントを失った場合はどうなりますか？

JWKSの所有者アカウントへのアクセスを失うと、JWKSを更新できなくなります。これにより、キーローテーションが発生した場合にフェデレーテッドキーレスアカウントが機能しなくなる可能性があります。アカウントの秘密鍵を安全に保管し、バックアップを作成することが重要です。

## トラブルシューティング

### トランザクションの署名に失敗する場合はどうすればよいですか？

以下を確認してください：
1. JWKSが正しく登録されているか
2. JWTトークンが有効か
3. 一時的な鍵が期限切れでないか

### JWTトークンの検証に失敗する場合はどうすればよいですか？

以下を確認してください：
1. IAMプロバイダーの設定が正しいか
2. JWKSが最新か
3. トークンが有効期限内か

### エラーメッセージの意味を教えてください

一般的なエラーメッセージとその解決方法：

- `OIDC_PROVIDER_ERROR`: IAMプロバイダーの設定を確認してください
- `GUARDIAN_ERROR`: ガーディアンサービスの状態を確認してください
- `PROVER_ERROR`: プルーバーサービスの状態を確認してください
- `NETWORK_ERROR`: ネットワーク接続を確認してください
- `EPHEMERAL_KEY_ERROR`: 一時的な鍵を再生成してください
