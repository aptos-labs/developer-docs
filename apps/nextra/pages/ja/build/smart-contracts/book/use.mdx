# use宣言

`use`宣言は、モジュールやその要素を現在のスコープにインポートするために使用されます。

## 基本的な使用方法

モジュール全体をインポートする：

```move
use 0x1::vector;
use std::string;
```

## エイリアスの使用

`as`キーワードを使用して、インポートしたモジュールや要素に別名を付けることができます：

```move
use 0x1::vector as vec;
use std::string::String as Str;
```

## 特定の要素のインポート

モジュールから特定の要素をインポートする：

```move
use std::vector::{empty, push_back, pop_back};
use aptos_framework::coin::{Self, Coin};
```

## 複数のインポート

複数のモジュールを一度にインポートする：

```move
use std::{vector, string, signer};
use aptos_framework::{
    coin::{Self, Coin},
    account,
    timestamp,
};
```

## インポートの可視性

`use`宣言は、以下の場所で使用できます：

- モジュールのトップレベル
- 関数内
- スクリプト内

```move
module 0x42::example {
    // モジュールレベルのインポート
    use std::vector;

    public fun example_function() {
        // 関数レベルのインポート
        use std::string;
        // ...
    }
}

script {
    // スクリプトレベルのインポート
    use std::debug;
    
    fun main() {
        // ...
    }
}
```

## テスト専用のインポート

テストコードでのみ使用するインポートには、`#[test_only]`アノテーションを使用します：

```move
#[test_only]
use std::unit_test;

#[test_only]
use 0x42::test_helpers;
```

## 例：完全な使用例

以下は、様々なインポート方法を示す完全な例です：

```move
module 0x42::token {
    // 標準ライブラリからのインポート
    use std::{
        string::String,
        vector,
        signer,
    };
    
    // フレームワークからのインポート
    use aptos_framework::{
        coin::{Self, Coin},
        account,
        timestamp,
    };
    
    // エイリアスを使用したインポート
    use 0x42::utils as token_utils;
    
    // テスト専用のインポート
    #[test_only]
    use std::unit_test;
    
    struct Token has key {
        name: String,
        supply: Coin,
    }
    
    public fun create_token(
        account: &signer,
        name: String,
        initial_supply: u64,
    ) {
        // インポートしたモジュールの使用
        let addr = signer::address_of(account);
        let coin = Coin::mint(initial_supply);
        move_to(account, Token { name, supply: coin });
    }
    
    #[test]
    fun test_create_token() {
        // テストコード
    }
}
```

## ベストプラクティス

- 必要な要素のみをインポートする
- 適切なエイリアスを使用して名前の衝突を避ける
- インポートは論理的にグループ化する
- テスト専用のインポートには`#[test_only]`を使用する
- 関数内のインポートは必要な場合のみ使用する
