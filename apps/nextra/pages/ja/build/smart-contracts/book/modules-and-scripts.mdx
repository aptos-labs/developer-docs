# モジュールとスクリプト

Moveには2種類のプログラムがあります：**_モジュール_**と**_スクリプト_**です。モジュールは、構造体型とそれらの型に対する操作を行う関数を定義するライブラリです。構造体型はMoveの[グローバルストレージ](global-storage-structure.mdx)のスキーマを定義し、モジュール関数はストレージを更新するためのルールを定義します。モジュール自体もグローバルストレージに保存されます。スクリプトは、従来の言語の`main`関数に似た実行可能なエントリーポイントです。スクリプトは通常、グローバルストレージを更新する公開モジュールの関数を呼び出します。スクリプトは一時的なコードスニペットであり、グローバルストレージには公開されません。

Moveのソースファイル（または**コンパイル単位**）には、複数のモジュールとスクリプトを含めることができます。ただし、モジュールの公開とスクリプトの実行は別々のVM操作です。

## 構文

### スクリプト

<Callout type="info">
Moveスクリプトの公開と実行方法については、[Moveスクリプト](../scripts/script-tutorial.mdx)の例を参照してください。
</Callout>

スクリプトは以下の構造を持ちます：

```text
script {
    <use>*
    <constants>*
    fun <identifier><[type parameters: constraint]*>([identifier: type]*) <function_body>
}
```

`script`ブロックは、すべての[`use`](uses.mdx)宣言で始まり、その後に[定数](constants.mdx)、そして最後に主要な[関数](functions.mdx)宣言が続きます。
メイン関数は任意の名前を付けることができ（つまり、`main`という名前である必要はありません）、スクリプトブロック内で唯一の関数であり、任意の数の引数を持つことができ、値を返してはいけません。以下は、これらの各コンポーネントを含む例です：

```move
script {
    // stdという名前付きアカウントアドレスで公開されているdebugモジュールをインポートします。
    use std::debug;

    const ONE: u64 = 1;

    fun main(x: u64) {
        let sum = x + ONE;
        debug::print(&sum)
    }
}
```

スクリプトの機能は非常に限られており、フレンドの宣言、構造体型の定義、グローバルストレージへのアクセスはできません。その主な目的はモジュール関数を呼び出すことです。

### モジュール

モジュールは以下の構文を持ちます：

```move
module <address>::<identifier> {
    (<use> | <friend> | <type> | <function> | <constant>)*
}
```

ここで、`<address>`は有効な[名前付きまたはリテラルアドレス](address.mdx)です。

例：

```move
module 0x42::example {
    struct Example has copy, drop { i: u64 }

    use std::debug;
    friend 0x42::another_example;

    const ONE: u64 = 1;

    public fun print(x: u64) {
        let sum = x + ONE;
        let example = Example { i: sum };
        debug::print(&sum)
    }
}
```

`module 0x42::example`の部分は、モジュール`example`が[グローバルストレージ](global-storage-structure.mdx)の[アカウントアドレス](address.mdx)`0x42`の下で公開されることを指定します。

モジュールは[名前付きアドレス](address.mdx)を使用して宣言することもできます。例：

```move
module example_addr::example {
    struct Example has copy, drop { a: address }

    use std::debug;
    friend example_addr::another_example;

    public fun print() {
        let example = Example { a: @example_addr };
        debug::print(&example)
    }
}
```

名前付きアドレスはソース言語レベルとコンパイル中にのみ存在するため、名前付きアドレスはバイトコードレベルでその値に完全に置き換えられます。例えば、以下のようなコードがある場合：

```move
script {
    fun example() {
        my_addr::m::foo(@my_addr);
    }
}
```

`my_addr`を`0xC0FFEE`に設定してコンパイルすると、操作的には以下と同等になります：

```move
script {
    fun example() {
        0xC0FFEE::m::foo(@0xC0FFEE);
    }
}
```

ただし、ソースレベルでは、これらは_同等ではありません_—関数`m::foo`は、そのアドレスに割り当てられた数値ではなく、`my_addr`名前付きアドレスを通じて_必ず_アクセスする必要があります。

モジュール名は`a`から`z`または`A`から`Z`の文字で始めることができます。最初の文字の後、モジュール名にはアンダースコア`_`、`a`から`z`の文字、`A`から`Z`の文字、または`0`から`9`の数字を含めることができます。

```move
module my_module {}
module foo_bar_42 {}
```

通常、モジュール名は小文字で始まります。`my_module`という名前のモジュールは、`my_module.move`という名前のソースファイルに保存する必要があります。

`module`ブロック内のすべての要素は任意の順序で表示できます。
基本的に、モジュールは[`型`](structs-and-resources.mdx)と[`関数`](functions.mdx)のコレクションです。
[`use`](uses.mdx)キーワードは他のモジュールから型をインポートするために使用されます。
[`friend`](friends.mdx)キーワードは信頼されたモジュールのリストを指定します。
[`const`](constants.mdx)キーワードはモジュールの関数で使用できるプライベート定数を定義します。
