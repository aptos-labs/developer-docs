# Move 2 リリースノート

このページではMove 2言語のリリースについて説明します。新機能のリファレンスドキュメントはこの本に統合されており、テキスト内で「_言語バージョン2.n以降_」と記されています。

## Move 2.1

Move 2.1言語リリースでは、以下の機能が追加されました：

- **複合代入** `x += n`、`x -= n`などを使用して、代入と算術演算を組み合わせることができるようになりました。サポートされている演算については[こちらのリファレンスドキュメント](variables.mdx#compound-assignments)を参照してください。

- **ループラベル** ループにラベルを付け、`break`や`continue`式でそのラベルを参照できるようになりました。これにより、ネストされたループの内部から外部ループを継続または中断することができます。詳細は[こちらのリファレンスドキュメント](loops.mdx#loop-labels)を参照してください。

- **アンダースコア関数パラメータはワイルドカード** `_`という名前の関数パラメータは、もはや変数のように動作しません：値をバインドせず、同じ関数に複数の`_`パラメータがあってもコンフリクトを引き起こしません。値式で`_`を使用するとエラーになります。これにより、`_`の動作がパターンやlet式でのワイルドカードとしての動作に近づきました。

## Move 2.0

Move 2.0言語リリースでは、以下の機能が追加されました：

- **列挙型** 1つの格納可能な型で異なるデータレイアウトのバリアントを定義するオプションを追加しました。[列挙型のセクション](enums.mdx)で説明されています。

- **レシーバースタイル関数** `value.func(arg)`という馴染みのある記法で関数を呼び出す機能を追加しました。[このセクション](functions.mdx#dot-receiver-function-call-style)で説明されています。

- **インデックス記法** `&mut vector[index]`や`&mut Resource[addr]`のような記法で[ベクターの要素](vector.mdx#index-notation-for-vectors)や[リソースストレージ](global-storage-operators.mdx#index-notation-for-storage-operators)にアクセスできるようになりました。

- **位置指定構造体** `struct Wrapped(u64)`のようなラッパー型を定義できるようになりました。位置指定構造体は[こちら](structs-and-resources.mdx#positional-structs)で説明されています。列挙型のバリアントも位置指定が可能です。

- **ドット2つのパターンワイルドカード** `let Struct{x, ..} = value`のような文でデータの一部を選択的にマッチできるようになりました。[こちら](structs-and-resources.mdx#partial-patterns)で説明されています。これらのパターンは列挙型のバリアントでも使用できます。

- **パッケージ可視性** 関数をパッケージ内のどこからでもアクセス可能（ただしパッケージ外からはアクセス不可）に宣言できるようになりました。フレンド関数も引き続きサポートされていますが、多くの場合パッケージ可視性の方が適しています。より簡潔な記法として、パッケージ関数とフレンド関数は`public(package) fun`や`public(friend) fun`の代わりに、単に`package fun`や`friend fun`と宣言できます。この機能は[こちら](functions.mdx#package-visibility)で説明されています。

- **アサートのアボートコードがオプションに** `assert!`マクロは1つの引数だけで使用できるようになり、アボートコードを省略できます。その場合、デフォルトのコードが使用されます。[こちら](abort-and-assert.mdx#assert)も参照してください。

- **新しいキャスト構文** これまでキャストは常に括弧で囲む必要があり、`function((x as u256))`のように書く必要がありました。この要件は廃止され、`function(x as u256)`のように括弧なしでキャストを最上位の式として書けるようになりました。ただし、式の中では`(x as u64) + (y as u64)`のように書く必要があります。これは新しい列挙型のテスト`data is VersionedData::V1`にも同様に適用されます。

- **評価順序の明確化** 以下のケースで評価順序が明確に定義されました（これまでは未定義でした）：
    - (a) 関数呼び出しの引数と、(b) 二項演算のオペランド式は、どちらも左から右の順序で評価されます。
    - `*lexp = rexp`という形式の「変更」式（[参照を通じた変更](variables.mdx#mutating-through-a-reference)を参照）において、`lexp`は型`&mut T`の式、`rexp`は型`T`の式です。`rexp`が最初に評価され、その後`lexp`が評価されます。

- **acquires注釈のバグ修正** [関数は`acquires`で注釈を付ける必要があります](functions.mdx#acquires)。これは、関数が`move_from`、`borrow_global`、または`borrow_global_mut`を使用してリソースにアクセスする場合（直接または他の関数を通じて間接的に）に必要です。そうでない場合はエラーとなります。以前は、遷移的な呼び出しグラフにサイクルが含まれている場合、このようなエラーは報告されませんでした：これは正しくない動作でした。遷移的な呼び出しグラフにサイクルがある場合でもこれらのエラーを報告するように修正しました。
