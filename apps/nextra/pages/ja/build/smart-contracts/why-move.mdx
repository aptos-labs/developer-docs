# なぜMove？

Moveプログラミング言語は、もともとFacebookのエンジニアチームによってDiem Payment Network向けに作成されました。
Moveは、プラットフォームに依存しない言語として設計され、大きく異なるデータモデルと実行モデルを持つ
様々なブロックチェーン間で、共通のライブラリ、ツール、開発者コミュニティを実現することを可能にします。
Aptosでは、Moveで強力な開発者コミュニティを構築し、Move on Aptosのスタック上に構築し、
オープンソースソフトウェアに貢献することを推奨しています。

Moveは以下の原則に基づいて構築されています：

| **原則**                   | **説明**                                                                                                                                                                                                                                                                                                                                                                   |
|----------------------------|-----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| **デフォルトで安全**      | 金融システムはユーザーが資金を失わないように構築されています。Moveは、リエントランシー攻撃、二重支払い、算術オーバーフローなどの攻撃やバグのクラス全体を防ぐように設計されています。型安全性とコンパイル時チェックがセキュリティの最前線にあります。                                                                                                                    |
| **ランタイム検証**        | バイトコードは実行時に検証され、問題が発生していないことを確認し、追加の安全性を提供し、悪意のある行為者を防ぎます。                                                                                                                                                                                                                                                    |
| **形式検証**              | Move on Aptosは、契約の形式検証を提供する仕様言語を提供します。これにより、不変条件の証明とコード監査の支援が可能になります。                                                                                                                                                                                                                                            |
| **シンプルさ**            | コマンドとバイトコードは意図的にシンプルです。これにより、デコンパイル、ランタイム検証、コード検査が容易になります。ブロックチェーン用の通常のプログラミング言語を使用する場合、スマートコントラクトに適合させるために言語の大部分を無視する必要があることがよくあります（例：Rust）。                                                                                |


## なぜAptos上のMove？

Aptos上のMoveは、Facebookのチームによって構築された完全な言語をサポートし、
セキュリティと開発者エクスペリエンスを向上させるための追加の拡張機能を備えています。

### セキュリティ

| **利点**                   | **説明**                                                                                                                                                                                                                                          |
|-----------------------------|----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| **形式検証**               | Aptosフレームワークは完全に仕様化され、Move Proverで形式的に検証されています。これにはガバナンス、NFT、トークンに関する主要な契約が含まれます。                                                                                                   |
| **ガスカバレッジ**         | Move VMは100%のガスカバレッジを持っています。ガスはシステムでの実際の使用量（CPU、メモリ、ストレージ、I/O）に基づいて課金されます。つまり、ガスの悪用はありません。                                                                              |
| **セキュリティ冗長性**     | ランタイム安全性チェックによって提供されるセキュリティ冗長性。                                                                                                                                                                                    |
| **権限制御**               | 権限制御は様々なレベルで柔軟に構築できます。例えば、RWAトークン化を可能にするために、デフォルトでトークンレベルの権限制御が存在します。                                                                                                         |


### 開発者エクスペリエンス

| **利点**                   | **説明**                                                                                                                                                                                                                                                                                                                                                                      |
|-----------------------------|--------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| **Move開発ツール**         | - **ユニットテスト**: Moveにはすべての契約に対する組み込みのユニットテストがあります。Aptosはさらに異なるシナリオをテストするためのフレームワークでのテスト機能を提供します。<br/> - **カバレッジ**: カバレッジツールはソースレベルとバイトコードレベルの両方でカバレッジレポートを提供します。<br/> - **デコンパイラ**: セキュリティ向上のため、オンチェーンのバイトコードは逆アセンブルまたはデコンパイルして実際の契約の可視性を提供できます。<br/> - **IDEプラグイン**: Aptosは2つの主要なIDEをサポートしています：[Visual Studio Code](https://marketplace.visualstudio.com/items?itemName=MoveBit.aptos-move-analyzer)と[IntelliJ](https://plugins.jetbrains.com/plugin/14721-move-on-aptos)。 |
| **データモデル**           | Aptosはデータ定義がオンチェーンに保存されるアクセス可能なデータモデルを持っています。オブジェクトとアカウントは解析しやすい形式で複数の異なる構造を持つことができます。                                                                                                                                                                                                   |
| **アップグレード可能性**   | アップグレード可能性により、アプリケーションインターフェースが破壊されることがなく、下流のアプリケーションからの明示的な採用を必要としません。バグを修正するために契約を単純にその場でアップグレードできます。                                                                                                                                                            |
| **クロス相互作用**         | Moveは型安全な構造体を使用して契約間の相互作用を可能にします。                                                                                                                                                                                                                                                                                                            |
| **コードストレージ**       | Aptosはソースコードをオンチェーンに保存し、契約とバイトコードの正確性を監査し確保する能力を向上させます。                                                                                                                                                                                                                                                                 |
| **スポンサード取引**       | ネイティブのスポンサード取引サポートにより、特別なサービスや契約固有のコードを必要とせずに、他のユーザーが取引を支払うことができます。                                                                                                                                                                                                                                  |
| **堅牢なトークン標準**     | デジタルアセットと代替可能アセット標準は、オンチェーン上の多様な種類のトークンとデジタルアセットに対して柔軟性と統一された標準を提供します。これらはERC-20、ERC-721、ERC-1155、Token-2022などの既存の標準の影響を受けています。                                                                                                                                      |
| **オンチェーンランダム性** | ネイティブのオンチェーン偏りのないランダム性は、コンパイル時の追加の安全性チェックを備えた、安全で一貫した乱数取得方法を提供します。                                                                                                                                                                                                                                    |
